  • Git Hook

    简介

    Git命令触发的时候, 执行特定操作(Shell脚本), 分为客户端的Hook和服务端的Hook, 此处介绍客户端Hook。

    客户端Hook种类

    种类 含义 用途
    pre-commit 在用户尝试提交更改之前运行 常用于执行代码格式化、代码质量检查等操作，确保提交的代码符合团队规范。
    prepare-commit-msg 在生成提交信息（即打开文本编辑器让用户输入提交信息）之前运行 可以用来自动填充提交信息模板，或者根据当前的变更情况预填充一些信息。
    commit-msg 在用户完成提交信息的编辑后，但在实际创建提交对象之前运行。 通常用于验证提交信息是否符合特定格式或包含必要的关键字。
    post-commit 在提交完成后运行 可以用来发送通知邮件、更新统计信息等。
    pre-rebase 在变基操作前运行 可以用来检查是否有未提交的更改，或者在变基前执行其他准备工作。
    pre-push 在用户尝试将本地分支推送到远程仓库之前运行 常用于执行更严格的代码审查，比如通过持续集成系统检查构建是否成功，或者检查是否有未解决的问题。

    如何配置

    配置在哪

    • git 分为全局配置和各个项目的配置,
    • 其中全局配置在家目录下的.git文件夹,
    • 各个项目的配置在项目中的.git文件夹

    配置流程

    1. 需要自己建一个文件夹myHooks(名称任意), 用于存放你的hook脚本

    2. 使用命令 git config --global core.hooksPath ./myHooks 配置全局Hook，或者使用 git config core.hooksPath ./myHooks 为当前项目指定Hook

    3. 上述配置步骤，也可以直接打开全局/项目中的.git目录, 找到 core.hooksPath 配置对应的Hooks存放目录

    4. 其中Hook脚本文件夹myHooks中的脚本, 必须严格按照上面种类命名, 比如 pre-commit

    5. 在响应的仓库, 执行对应的git命令, 可以看到脚本执行
